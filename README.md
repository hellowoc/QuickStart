# QuickStart — 实习生入职系统

> 欢迎！如果你是新人，按下面的步骤走，AI 会在每个阶段协助你。

## 第一步：环境准备（第0天）

```bash
git clone https://github.com/hellowoc/QuickStart.git
cd QuickStart
bash install.sh
```

安装脚本会自动：
- 部署 onboarding skill
- 启动个人信息面试（AI 问你答）
- 生成 `profile.local.md`
- 创建你的个人知识目录 `~/mddata_你的名字/`

安装完成后重启 Claude Code，对 AI 说：**"带我认识 QuickStart"**

---

## 第二步：认识知识库（第1-3天）

对 AI 说：**"带我认识 ZKDataBase"**

AI 会引导你：
1. 阅读 `_主线_/` 下的工程概述，建立全局认知
2. 了解 `_模块_/` 的目录结构
3. 明白三区架构（主线 / 模块 / 个人）的使用规则

看完后对 AI 说：**"搭建开发环境"**，AI 会根据知识库中的工程信息指导你。

---

## 第三步：选择负责模块（第1周）

对 AI 说：**"帮我选择负责的模块"**

AI 会根据你的：
- profile.local.md 中的技术栈
- ZKDataBase 中已有的模块文档
- 导师的分配建议

推荐合适的起步模块。选定后把模块名写入 `~/mddata_你的名字/学习路线.md`。

---

## 第四步：开始深度学习（第1-4周）

对 AI 说：**"开始学习 [模块名]"**

AI 按三层教学法展开：
1. 基础零件清单 → 查漏补缺
2. 主文档 + 基础知识文档 → 对照阅读
3. 界面控件溯源 → 逐行追踪

每学完一个模块，在 `~/mddata_你的名字/模块笔记/` 下产出自己的理解。

---

## 第五步：独立负责（第5-12周）

开始独立处理模块相关的问题和需求。过程中：
- 踩坑 → 记入 `~/mddata_你的名字/踩坑记录.md`
- 有价值的理解 → 署名发布到 ZKDataBase 的 `_模块_/` 目录

对 AI 说：**"我要往知识库发布一篇模块文档"**，AI 会帮你按格式整理。

---

## 第六步：归档 & 考核（第12周）

对 AI 说：**"实习期结束，帮我归档"**

AI 会自动：
1. 扫描你 `~/mddata_你的名字/` 下的所有产出
2. 标出有价值回流入主线的内容
3. 生成《归档建议》，提交给导师评审
4. 对照《实习考核清单》逐项自评

导师审核通过后，你的署名文档正式进入 ZKDataBase `_模块_/`，精华部分并入 `_主线_/`。

---

## 全程 AI 辅助

| 你想做什么 | 对 AI 说 |
|-----------|---------|
| 了解工程整体 | "带我认识 ZKDataBase" |
| 选模块 | "帮我选择负责的模块" |
| 学模块 | "开始学习 [模块名]" |
| 深入某个点 | "细讲这个" |
| 停下来消化 | "停，我消化一下" |
| 临时问个无关问题 | "题外话：[问题]" |
| 发布署名文档 | "我要往知识库发布一篇模块文档" |
| 实习结束归档 | "实习期结束，帮我归档" |
