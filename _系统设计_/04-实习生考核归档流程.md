# 实习生考核归档流程

## 归档时机

实习期最后一周，实习生对 AI 说：**"实习期结束，帮我归档"**

## AI 自动归档流程

### 第一步：扫描个人产出

AI 扫描 `~/mddata_xxx/` 下的所有内容：
- 模块笔记（数量、覆盖模块、深度）
- 踩坑记录（问题数、解决数、可复用度）
- 署名文档（已发布的）

### 第二步：生成归档建议

AI 对每篇产出标注回流级别：

| 级别 | 标准 | 去向 |
|------|------|------|
| S | 补充了主线缺失的关键信息 | `_主线_/` |
| A | 对前人文档有实质性深化 | `_模块_/`，加"_权威"前缀 |
| B | 有价值的个人视角，供参考 | `_模块_/`，保留署名 |
| C | 个人学习笔记，不回流 | 仅保留在 `_归档_/` |

### 第三步：导师评审

1. AI 生成《归档建议》文档
2. 实习生提交给导师
3. 导师勾选同意回流的内容
4. 导师填写《实习考核清单》总评

### 第四步：执行归档

```
导师确认后：

1. S级内容 → 合并到 _主线_/ 对应文件
2. A/B级内容 → 复制到 _模块_/ 对应目录
3. 实习生产出 → 打包到 _归档_/实习生名字/
4. 原个人目录 ~/mddata_xxx/ 保留不删
```

## 对下一届的影响

归档完成后，下一届新人：
1. 能在 `_模块_/` 中看到往届署名文档
2. 能在 `_归档_/` 中参考往届完整学习路径
3. `_主线_/` 持续完善，越来越权威
