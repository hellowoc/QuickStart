# 新人入职四阶段流程

## 前置：安装 skill

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/hellowoc/Claude_Memory/main/.claude/skills/onboarding/install.sh)"
# 然后执行 /reload-skills
```

## 第0步：个人信息面试（必须，入职当天）

对着 AI 说："帮我完成入职信息录入"

AI 逐一询问：
1. 怎么称呼你？（如 张工、小李）
2. 教育背景？（学校、专业、学历）
3. 技术栈水平？（C/C++/Qt/Linux/网络/调试 各自评级）
4. 你习惯怎么学新代码？（先看文档 / 先跑起来 / 先看结构图）
5. 和 AI 交流偏好？（简短直给 / 详细解释 / 先框架后细节）

→ 自动生成 `profile.local.md`

## 第1天：环境 + 概览

1. 根据 skill 中的工程信息搭开发环境
2. 打开 `mdData/README.md` 了解知识库结构
3. 跟着 `_主线_/工程概述.md` 建立整体认知

## 第1周：跟模块

1. 选择要跟的模块（由导师指定或自己选）
2. 让 AI 按三层教学法讲解（skill 中已定义）
3. 产出写入 `_个人_/你的名字/`：
   - `学习路线.md`：自己定计划
   - `模块名-我的理解.md`：自己的笔记

## 第1月：独立负责

1. 独立负责一个模块
2. 踩坑写入 `_个人_/` 或整理后署名发到 `_模块_/`
3. 发现的代码逻辑关系补充到 `_模块_/`

## 转正期：知识回流

1. 回顾 `_个人_/` 下的笔记
2. 提炼"对下一届有用的"提交 PR
3. 更新负责模块信息到索引
