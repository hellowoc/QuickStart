# 环境变量配置-基础知识

本文档是 [[环境变量配置-主文档]] 的配套基础知识文档。

---

## 1. getenv() 和 setenv()

C 标准库提供的环境变量读写函数，声明在 `<stdlib.h>`：

```c
char *getenv(const char *name);
// 返回值：指向环境变量值的指针（不可修改，修改会影响后续 getenv 调用）
// 找不到返回 NULL

int setenv(const char *name, const char *value, int overwrite);
// overwrite=0：如果变量已存在则不覆盖
// overwrite=1：如果变量已存在则覆盖
// 返回值：0 成功，-1 失败（errno 设置为 ENOMEM）
```

## 2. export（shell 内建命令）

`export` 不是可执行文件，是 shell 的内建命令。作用是将一个变量标记为"传递给子进程"。

```bash
FOO=bar           # 普通变量，只有当前 shell 能看到
export FOO=bar    # 导出变量，当前 shell 启动的子进程也能看到
```

子进程的环境变量表是从父进程拷贝的——`export` 就是把变量放进这个表的操作。

## 3. $ORIGIN 变量

动态链接器识别的特殊变量，展开为可执行文件所在目录的绝对路径。

```bash
gcc main.o -L. -lhello -Wl,-rpath,'$ORIGIN/../lib' -o main
```

`$ORIGIN` 由动态链接器在运行时展开，不是 shell 变量。所以编译时必须用单引号包住，防止 shell 提前展开。

假设程序安装路径为 `/opt/myapp/bin/main`：
- `$ORIGIN` = `/opt/myapp/bin`
- `$ORIGIN/../lib` = `/opt/myapp/lib`
- 动态链接器去 `/opt/myapp/lib` 找 .so

## 4. .bashrc 和 .bash_profile

| 文件 | 何时执行 | 用途 |
|------|---------|------|
| `~/.bashrc` | 每次打开交互式终端（非登录 shell） | 放 alias、export、PS1 等 |
| `~/.bash_profile` | 登录时（通过 ssh 或 tty 登录） | 放一次性初始化 |

环境变量通常写进 `~/.bashrc`，因为每次打开新终端都需要它们。

加载修改后的配置：

```bash
source ~/.bashrc
# 或
. ~/.bashrc
```

## 5. ldconfig 缓存 vs LD_LIBRARY_PATH 的优先级

关键在于搜索顺序：

```
RPATH → RUNPATH → LD_LIBRARY_PATH → ldconfig缓存 → 系统默认路径
```

LD_LIBRARY_PATH 排在 ldconfig 缓存前面——即使你用 `ldconfig` 更新了系统缓存，LD_LIBRARY_PATH 中的同名 .so 仍会被优先加载。这是最常见的"明明装了新库但程序还是加载旧库"的原因。

排查命令：

```bash
# 看程序实际加载了哪个 .so
ldd ./main

# 看某个 .so 提供了哪些符号
nm -D /path/to/lib.so
```

## 6. CMAKE_PREFIX_PATH 和 CMake 的 find 机制

`find_package()` 的搜索路径优先级：

1. CMake 变量 `CMAKE_PREFIX_PATH`
2. 环境变量 `CMAKE_PREFIX_PATH`（注意：和环境变量拼写相同但作用域不同）
3. CMake 内置的系统路径（`/usr`、`/usr/local`）

推荐用 CMake 参数而非环境变量——参数是显式的，写进构建脚本或命令行，换机器不会丢失。

```bash
# CMake 参数方式（推荐）
cmake .. -DCMAKE_PREFIX_PATH=/home/user/my_libs

# 环境变量方式（不推荐）
export CMAKE_PREFIX_PATH=/home/user/my_libs
cmake ..
```
