# ZKDataBase 使用指南

> 这是 CameraTest 色选机工程的技术知识库。三区结构，各司其职。

## 三区结构

```
ZKDataBase/
├── _主线_/      ← 权威参考，新人从这里开始
├── _模块_/      ← 协同贡献，署名文档
└── _归档_/      ← 往届实习生完整产出
```

## _主线_/ — 权威参考区

新人第一眼看的。按模块编排，一条干净的主线：

| 阅读顺序 | 文档 | 内容 |
|---------|------|------|
| 1 | 工程概述.md | 整体架构、核心数据流 |
| 2 | 通信层-串口协议.md | 自定义二进制帧、CMD命令字 |
| 3 | 通信层-UDP三通道.md | IPC/FPGA/高速抓图三通道 |
| 4 | 通信层-MQTT全链路.md | 协议→嵌入式部署→加密体系 |
| 5 | 数据流-核心链路.md | 相机→FPGA→界面 全链路 |
| 6 | 多线程架构.md | 7-9个常驻线程的协调 |
| 7 | references/ | 基础零件字典（Qt类型、Socket API等） |

主线由知识库维护者负责更新。

## _模块_/ — 协同贡献区

任何人可以署名发布的深入文档。命名规则：

- `_权威-xxx.md` — 维护者写的延伸文章，与主线等权威
- `张三-xxx.md` — 个人署名贡献，读者自行判断可信度

同一个主题允许多个署名文章并存，多角度互补。

## _归档_/ — 往届实习生产出

每届实习生的完整学习产物打包存放。供后续新人参考学习路径。

## 如何使用

1. **新人**：按 `_主线_/` 阅读顺序建立骨架 → 根据负责模块深入 `_模块_/`
2. **实习生**：产出先进个人目录，有价值的署发布到 `_模块_/`
3. **维护者**：定期审视 `_模块_/`，将经过验证的精华并入 `_主线_/`
