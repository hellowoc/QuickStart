# 学习风格

## 学习的核心模式：广度遍历 → 找关系 → 梳理逻辑 → 按框架深挖

用户的学习过程分为四个阶段：

1. **广度扫描**：先把一个知识领域（如 MQTT）的所有相关概念铺开看一遍，不做筛选和过滤，全部收进来
2. **发现关系**：通过追问概念之间的关系（"QoS 和 TCP 握手是什么关系？"、"TLS 和 OpenSSL 是什么关系？"）把散落的知识点连成网
3. **自己的逻辑框架**：基于找出的关系，梳理出一套自己的学习路线——不是文档给的顺序，而是自己推导出来的逻辑顺序
4. **按框架深挖**：按照这套自建逻辑，逐一深入每个知识点，直到完全掌握

## 阅读工程的逻辑习惯

- **广度优先**：先全面扫描整个知识领域，了解所有相关概念的存在
- **寻找关系**：不满足于孤立的知识点，主动追问概念之间的关联
- **自建逻辑**：不被动接受文档给定的顺序，自己重新组织学习路线
- **逐层拆解**：对每个函数要求追溯到最底层的系统调用或 Qt API
- **串联理解**：不喜欢孤立的文件解读，要求把跨文件的机制串联
- **先框架后细节**：先要整体路线图，再逐个模块深入
- **要求 md 导出**：每次解释后默认导出 md 文档到指定文件夹，作为外部记忆

## 提问模式

- 对陌生的底层概念会立刻提问（JSON、SQL、锁、QPainter...）
- 用"我是否可以理解为..."句式验证自己的理解
- 困惑时会精确指出困惑点（如"为什么删两次文件"）
- 不满足于"它是干什么的"，会追问"为什么这样设计"
- 进入完全陌生领域时会诚实暴露盲区（"我一窍不通，甚至不知道怎么提问"）
- 使用约定的快捷信号：题外话（简单概念不参与主流程）、暂停、深度、跳过
- 阶段结束后主动要求总结分析，反思学习方法

## 对 AI 的使用偏好

- 用 AI 替代文档阅读——不自己看 Qt 官方文档，通过问答学习
- 用 md 导出替代笔记
- 按预定路线逐步推进，不被无关细节拉走
- 需要先确认整体框架再深入
- 全新领域采用"框架优先"策略：先要概念地图，再按需深入
- 用 Obsidian 管理 mdData/ 知识库，用 `[[文件名]]` 交叉链接
- 阶段学习结束后要求总结分析（多维度的学习方法/技术认知/思维方式评估）
