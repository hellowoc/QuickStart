# 快捷交流信号

## 题外话

用户说"题外话"时，表示接下来的内容是一个简单概念性问题，只需要简短回答，**不参与主学习流程**。不需要导出MD，不需要展开讲解，不需要关联到当前学习主题。

**Why:** 在学习过程中会突然想到一些零散的概念问题，这些问题不应该打断主学习流程或导致文档导出。

**How to apply:** 用户以"题外话"开头的问题 → 简短回答（2-3句话即可）→ 不导出MD → 不关联主学习流程 → 等待用户回到主流程。

## 暂停信号

用户说"停，我消化一下"或类似表达时，立即停止输出新信息，帮助回顾和整理已讲过的内容。

**Why:** 信息量过大时用户需要时间消化，强制暂停避免信息过载。

## 深度信号

用户说"深入这个"或"细讲这个"时，从当前停留的知识点继续往下挖，不按原计划推进到下一个主题。

**Why:** 逐层深入的学习方式下，用户可能需要在某一层停留更久。

## 跳过信号

用户说"这个先跳过"时，跳过当前话题，继续下一个。

**Why:** 对当前目标不重要的内容可以后续再学，节省时间。

## 代办信号

用户说「代办」或「先记下来」时，进行以下操作

### 行为

1. 用户说关键字时，把当前问题或任务存入本文件，标注「待完成」
2. **后续每次对话开始时，主动提醒用户当前有多少个待完成的代办**，并列出标题
3. 用户说「跳过 XX」或「取消 XX」时，从列表中移除
4. 用户完成后，将状态改为「已完成」并记录完成日期

**Why:** 学习过程中会突然冒出和当前主题无关的问题/想法，如果立刻切换会打断主流程，如果不记下来会忘记。灵感暂存让这些想法不丢失也不打断学习。

**How to apply:** 用户说「灵感」→ 将问题存入本文件的待解决列表 → 每次对话开始时列出提醒 → 用户说「取消/跳过」才停止提醒。
