本文件是本人在使用ai辅助阅读工程文件时产生的记忆文件

MEMORY：
顶层索引，
内涵各个记忆文件的核心内容和地址索引，ai可以根据该文档定位个人记忆库
使用者可以让ai在此基础上进行记忆的修改，将本文件和ai本地的记忆文件进行同步
从而让不同机器上的ai有着相同的工作习惯

communication_signals：
本人的交流习惯
包含了一些快捷的操作指令，
使用者可以加入自己使用ai时的习惯，构建自己高效的交流逻辑

debug_experience：
调试过程中的记忆文件，
包含了多端调试的内容以及本人的调试习惯，
使用者可以在该记忆中记录调试偏好，如使用到的交叉工具编译链，工程中常见问题的修改方法等内容，
从而形成自己的调试助手

feedback_preferences：
ai反馈偏好
使用者可以加入ai给与反馈的时候的偏好设置
其中添加的“哥哥称呼”用于提醒ai使用者，当前窗口是否遗失了记忆，出现了ai幻觉，只要没叫哥哥，大概率就是出问题了

feedback_teaching_methodology：
学习时的要求
用户可以在本文件中加入ai教学时候的个各种要求

project_cameratest_reading：
包含了本工程的学习路径，以及工程的详细流程解读
ai可以通过该文件熟悉本工程，为ai指定源码路径之后，可以让ai直接熟悉你的工作工程版本
使用者可以用本文件来做自己工作版本的工程解读，让多端ai同步工程记忆
从而得到一个了解工程的工作协助者

user_xxx:
此部分为个人消息
使用者可以加入自己私人风格
